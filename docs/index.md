 Phase 5. Custom Project

 Basis and Data

For my custom project, I started with the three raw smart‑sales CSV files; customers_data.csv, products_data.csv, sales_data.csv

Customers; Duplicate CustomerID values, Inconsistant regions (east, EAST, East), GoldMember values in mixed formats.

Products; UnitPrice and WeightPounds stored as text, Category and Supplier values inconsistently formatted.

Sales; “?” characters in numeric fields, Missing PaymentMethod values, SaleAmount stored as text. Outliers in SaleAmount, CampaignID missing or non‑numeric, SaleDate stored as strings.

 Issues I chose to address
I focused on issues that directly affect BI analysis. Duplicates, Invalid characters, Missing values, Numeric conversion, Standardizing categories and regions, Outlier filtering, PaymentMethod cleanup.

 Issues
None!

 Assumptions
I assumed; “?” always indicates invalid data, Regions should be titled the same way, Missing CampaignID can be safely filled with 0, Median SaleAmount is a reasonable replacement for missing values.

 Changes
  The values that were targeted were different; sales data required more cleaning as it had missing values ingrained within the dataset. Customer data had to be cleaned to make sure that the data was easily identifiable - the data was input in different contexts and as a result was cleaned to ensure consistancy. Product data was focused towards categories providing easily parsable datasets.

 Verification
Visual inspection of charts, Confirming cleaned CSVs existed in data/clean.
